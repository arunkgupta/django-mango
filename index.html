About Django Mango

We don't have a proper definition for the project (yet) so instead I'll to explain what its for. Last June 30 to July 1 2012 was the first ever PyCon Philippines, with only a month preparation, some things weren't smooth like the registration process, proposal submission, volunteer registration, etc. django-mango aims to solve these problems by providing a platform to manage all the moving parts in organizing a conference.

How do I contribute?

We're passionate about opensource and as such, the project shares the same. However we need a way to organize contribution to the project so we agreed on the following guidelines:

  1. Fork the project at https://github.com/pypug/django-mango
  2. Commit your changes to your fork of the project and do a pull request
  3. Once your changes has been reviewed by the team, it'll be merged to the project

We trust everyone in the team, hence we trust their judgement against pull requests coming from contributors. At certain point in time, if the contributor earns a level of trust, we will add the user to the github organization to have a direct access on the project.

TIP

Your changes will have a greater success rate of being merged if your code is well documented and tested.
